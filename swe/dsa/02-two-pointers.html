Two Pointers

Converging and diverging pointer techniques that turn O(n²) brute force into O(n) elegance

TypeScript Pattern

The Two Pointer Technique

The two-pointer technique uses two indices that move through an array simultaneously, eliminating the need for nested loops. It's most commonly applied to sorted arrays or sequences where the relative position of elements matters.

The core insight: instead of checking every possible pair (O(n²)), we can use the sorted order to intelligently skip large portions of the search space. If the current sum is too small, move the left pointer right to increase it. Too large? Move the right pointer left to decrease it.

When to Reach for Two Pointers

Consider two pointers when you see: a sorted array, a problem involving pairs/triplets, palindrome checking, merging sorted arrays, or any problem where brute force would use nested loops over the same array.

Converging Pointers

The most common variant: start with one pointer at the beginning and one at the end, then move them toward each other until they meet. At each step, the decision of which pointer to move is driven by the problem's constraints.

Two Sum II (Sorted)

Given a 1-indexed sorted array and a target, find two numbers that sum to the target. Because the array is sorted, we don't need a hash map — two pointers give us O(1) space.

function twoSumSorted(nums: number[], target: number): [number, number] {
  let left = 0;
  let right = nums.length - 1;

  while (left < right) {
    const sum = nums[left] + nums[right];

    if (sum === target) {
      return [left + 1, right + 1]; // 1-indexed result
    }

    if (sum < target) {
      left++;    // Need a larger sum → move left pointer right
    } else {
      right--;   // Need a smaller sum → move right pointer left
    }
  }

  throw new Error('No solution found');
}
Why this works: Since the array is sorted, nums[left] is the smallest candidate and nums[right] is the largest. If their sum is too small, the only way to increase it is to pick a larger left value. If it's too large, pick a smaller right value. This guarantees we explore every relevant pair without nested loops.

Three Sum

Find all unique triplets that sum to zero. The strategy: sort the array, then for each element, run a two-pointer search on the remaining elements. Skip duplicates to avoid repeated triplets.

function threeSum(nums: number[]): number[][] {
  const result: number[][] = [];
  nums.sort((a, b) => a - b);

  for (let i = 0; i < nums.length - 2; i++) {
    // Skip duplicates for the first element
    if (i > 0 && nums[i] === nums[i - 1]) continue;

    // Early exit: if smallest possible sum > 0, no more triplets
    if (nums[i] > 0) break;

    let left = i + 1;
    let right = nums.length - 1;

    while (left < right) {
      const sum = nums[i] + nums[left] + nums[right];

      if (sum === 0) {
        result.push([nums[i], nums[left], nums[right]]);

        // Skip duplicates for both pointers
        while (left < right && nums[left] === nums[left + 1]) left++;
        while (left < right && nums[right] === nums[right - 1]) right--;

        left++;
        right--;
      } else if (sum < 0) {
        left++;
      } else {
        right--;
      }
    }
  }

  return result;
}

// threeSum([-1, 0, 1, 2, -1, -4])
// → [[-1, -1, 2], [-1, 0, 1]]

Duplicate Handling is Crucial

Without the duplicate-skipping logic, [-1, -1, 0, 1] would produce [-1, 0, 1] twice. Always skip adjacent equal elements after finding a valid triplet and when iterating the outer loop.

Container With Most Water

Given an array of heights, find two lines that together with the x-axis form a container holding the most water. The area between two lines at indices i and j is min(height[i], height[j]) × (j − i).

The greedy insight: start with the widest container (pointers at both ends). Then always move the shorter line inward — because moving the taller one can only decrease the width without any guarantee of increasing the height.

function maxArea(height: number[]): number {
  let left = 0;
  let right = height.length - 1;
  let max = 0;

  while (left < right) {
    const w = right - left;
    const h = Math.min(height[left], height[right]);
    max = Math.max(max, w * h);

    // Move the shorter line inward
    if (height[left] < height[right]) {
      left++;
    } else {
      right--;
    }
  }

  return max;
}

Fast and Slow Pointers

A variation where both pointers start at the same position but move at different speeds. The classic application is cycle detection in linked lists (Floyd's algorithm): the slow pointer moves one step at a time, the fast pointer moves two. If there's a cycle, they'll eventually meet.

// Preview: detecting a cycle in a linked list
function hasCycle(head: ListNode | null): boolean {
  let slow = head;
  let fast = head;

  while (fast?.next) {
    slow = slow!.next;
    fast = fast.next.next;
    if (slow === fast) return true;
  }

  return false;
}
