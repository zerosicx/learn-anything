Sliding Window

Efficiently processing contiguous subarrays and substrings by maintaining a moving window over the data

TypeScript Pattern

The Sliding Window Technique

Many problems ask you to find or compute something over contiguous subarrays or substrings. The brute-force approach — checking every possible subarray — runs in O(n²) or worse. The sliding window technique reduces this to O(n) by maintaining a window that slides across the data, reusing computation from the previous position.

The key insight: when the window moves one position right, you only need to add one new element and remove one old element, rather than recomputing from scratch.

When to use sliding window: The problem involves a contiguous sequence (subarray or substring), and you need to find a maximum, minimum, longest, shortest, or count that satisfies some condition. If "contiguous" is a requirement, sliding window is likely the right pattern.

Fixed vs Variable Window

There are two flavours of sliding window:

  • Fixed-size window: The window length is given (e.g., "find max sum of k consecutive elements"). You slide a window of exactly size k across the array.
  • Variable-size window: The window grows and shrinks to satisfy a condition (e.g., "find the smallest subarray with sum ≥ target"). You expand the right boundary to include more, then contract the left boundary to find the optimum.

Fixed-Size Window

The simplest variant: find the maximum sum of any subarray of exactly size k. Instead of summing k elements for each starting position (O(nk)), maintain a running sum and update it as the window slides.

function maxSumSubarray(nums: number[], k: number): number {
  if (nums.length < k) return 0;

  // Build the first window
  let windowSum = 0;
  for (let i = 0; i < k; i++) {
    windowSum += nums[i];
  }

  let maxSum = windowSum;

  // Slide: add the next element, remove the leftmost
  for (let i = k; i < nums.length; i++) {
    windowSum += nums[i] - nums[i - k];
    maxSum = Math.max(maxSum, windowSum);
  }

  return maxSum;
}

// maxSumSubarray([2, 1, 5, 1, 3, 2], 3) → 9 (subarray [5, 1, 3])

Each iteration does O(1) work — one addition and one subtraction — making the total time O(n).

Variable-Size Window

When the window size isn't fixed, you expand and contract it dynamically. The general template:

Variable Window Template

function variableWindow(input: string | number[]): number {
  let left = 0;
  let result = 0;  // or Infinity, depending on min/max

  for (let right = 0; right < input.length; right++) {
    // 1. Expand: add input[right] to window state

    // 2. Contract: while window is invalid, shrink from left
    while (/* window is invalid */) {
      // Remove input[left] from window state
      left++;
    }

    // 3. Update result with current valid window
    result = Math.max(result, right - left + 1);
  }

  return result;
}

The outer loop moves right forward. The inner while shrinks the window by advancing left until the window satisfies the constraint again. Despite the nested loop, each element is added and removed at most once, so the total work is O(n).

Longest Substring Without Repeating Characters

Given a string, find the length of the longest substring that contains no duplicate characters. This is the classic variable-window problem — expand right to include new characters, and contract left whenever a duplicate enters the window.

function lengthOfLongestSubstring(s: string): number {
  const seen = new Map<string, number>(); // char → most recent index
  let left = 0;
  let maxLen = 0;

  for (let right = 0; right < s.length; right++) {
    const char = s[right];

    // If char was seen and is within current window, jump left past it
    if (seen.has(char) && seen.get(char)! >= left) {
      left = seen.get(char)! + 1;
    }

    seen.set(char, right);
    maxLen = Math.max(maxLen, right - left + 1);
  }

  return maxLen;
}

// lengthOfLongestSubstring("abcabcbb") → 3 ("abc")
// lengthOfLongestSubstring("bbbbb")    → 1 ("b")
// lengthOfLongestSubstring("pwwkew")   → 3 ("wke")

Why a Map instead of a Set?

A Set only tells you whether a character is in the window. A Map storing the character's index lets you jump left directly to the position after the duplicate, avoiding a costly inner while loop that removes characters one by one.

Minimum Window Substring

Given strings s and t, find the smallest substring of s that contains all characters of t (including duplicates). This is one of the hardest sliding window problems because you need to track frequency counts, not just presence.

Strategy

  1. Build a frequency map of t.
  2. Expand the window right, decrementing the count for each character. When a character's count hits zero, you've satisfied that character.
  3. Once all characters are satisfied, contract from the left to find the minimum window.
  4. When contracting causes a character to become unsatisfied, stop and expand again.
function minWindow(s: string, t: string): string {
  if (t.length > s.length) return "";

  const need = new Map<string, number>();
  for (const c of t) {
    need.set(c, (need.get(c) ?? 0) + 1);
  }

  let required = need.size;  // distinct chars we still need
  let formed = 0;           // distinct chars fully satisfied
  const windowCounts = new Map<string, number>();

  let left = 0;
  let minLen = Infinity;
  let minStart = 0;

  for (let right = 0; right < s.length; right++) {
    const char = s[right];
    windowCounts.set(char, (windowCounts.get(char) ?? 0) + 1);

    if (need.has(char) && windowCounts.get(char) === need.get(char)) {
      formed++;
    }

    // Contract from the left while window is valid
    while (formed === required) {
      const windowLen = right - left + 1;
      if (windowLen < minLen) {
        minLen = windowLen;
        minStart = left;
      }

      const leftChar = s[left];
      windowCounts.set(leftChar, windowCounts.get(leftChar)! - 1);
      if (need.has(leftChar) && windowCounts.get(leftChar)! < need.get(leftChar)!) {
        formed--;
      }
      left++;
    }
  }

  return minLen === Infinity ? "" : s.slice(minStart, minStart + minLen);
}

// minWindow("ADOBECODEBANC", "ABC") → "BANC"
The "formed vs required" trick: Instead of comparing entire frequency maps on every iteration (which would be O(|t|)), we track how many distinct characters have been fully satisfied. Updating this count is O(1) per character.

Permutation in String

Given strings s1 and s2, return true if s2 contains a permutation of s1. A permutation is any rearrangement, so we're really asking: does s2 contain a contiguous substring with the same character frequencies as s1?

This is a fixed-size window (size = s1.length) with frequency matching.

function checkInclusion(s1: string, s2: string): boolean {
  if (s1.length > s2.length) return false;

  const k = s1.length;
  const s1Freq = new Array(26).fill(0);
  const winFreq = new Array(26).fill(0);

  const idx = (c: string) => c.charCodeAt(0) - 97;

  // Build frequency for s1 and the first window in s2
  for (let i = 0; i < k; i++) {
    s1Freq[idx(s1[i])]++;
    winFreq[idx(s2[i])]++;
  }

  let matches = 0;
  for (let i = 0; i < 26; i++) {
    if (s1Freq[i] === winFreq[i]) matches++;
  }

  for (let right = k; right < s2.length; right++) {
    if (matches === 26) return true;

    // Add new right character
    const rIdx = idx(s2[right]);
    winFreq[rIdx]++;
    if (winFreq[rIdx] === s1Freq[rIdx]) matches++;
    else if (winFreq[rIdx] === s1Freq[rIdx] + 1) matches--;

    // Remove leftmost character of previous window
    const lIdx = idx(s2[right - k]);
    winFreq[lIdx]--;
    if (winFreq[lIdx] === s1Freq[lIdx]) matches++;
    else if (winFreq[lIdx] === s1Freq[lIdx] - 1) matches--;
  }

  return matches === 26;
}

// checkInclusion("ab", "eidbaooo") → true ("ba" is a permutation of "ab")
// checkInclusion("ab", "eidboaoo") → false

The Matches Counter

Instead of comparing all 26 frequency slots each iteration, we maintain a matches counter that tracks how many character slots have equal frequency between s1 and the current window. When matches === 26, every character frequency matches — the window is a permutation.

Complexity Summary

Problem Time Space Window Type
Max Sum Subarray (size k) O(n) O(1) Fixed
Longest Substring No Repeat O(n) O(min(n, m)) Variable
Minimum Window Substring O(n + m) O(n + m) Variable
Permutation in String O(n) O(1) Fixed
Why sliding window is O(n): Each element is processed at most twice — once when right adds it and once when left removes it. Two passes through n elements is still O(n). The nested while loop is deceptive; it doesn't restart from the beginning each time.
