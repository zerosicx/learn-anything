What is a Tree?

A tree is a hierarchical data structure consisting of nodes connected by edges. Unlike linear structures (arrays, linked lists), trees branch outward, allowing one-to-many relationships. Every tree has a single root node, and every other node has exactly one parent.

Terminology

  • Root: The topmost node with no parent.
  • Leaf: A node with no children.
  • Height: The number of edges on the longest path from a node to a leaf. The tree's height is the height of its root.
  • Depth: The number of edges from the root down to a node. The root has depth 0.
  • Subtree: A node and all its descendants form a subtree — every node is the root of its own subtree.

Binary Tree Node

A binary tree is a tree where each node has at most two children, conventionally called left and right.

class TreeNode<T> {
  constructor(
    public val: T,
    public left: TreeNode<T> | null = null,
    public right: TreeNode<T> | null = null
  ) {}
}

Tree Traversals

Traversal means visiting every node exactly once. The three classic depth-first traversals differ only in when they process the current node relative to its subtrees.

Recursive Implementations

function inorder<T>(root: TreeNode<T> | null, result: T[] = []): T[] {
  if (!root) return result;
  inorder(root.left, result);
  result.push(root.val);
  inorder(root.right, result);
  return result;
}

function preorder<T>(root: TreeNode<T> | null, result: T[] = []): T[] {
  if (!root) return result;
  result.push(root.val);
  preorder(root.left, result);
  preorder(root.right, result);
  return result;
}

function postorder<T>(root: TreeNode<T> | null, result: T[] = []): T[] {
  if (!root) return result;
  postorder(root.left, result);
  postorder(root.right, result);
  result.push(root.val);
  return result;
}

Iterative Inorder (using a stack)

function inorderIterative<T>(root: TreeNode<T> | null): T[] {
  const result: T[] = [];
  const stack: TreeNode<T>[] = [];
  let curr = root;

  while (curr || stack.length) {
    // Go as far left as possible
    while (curr) {
      stack.push(curr);
      curr = curr.left;
    }
    curr = stack.pop()!;
    result.push(curr.val);
    curr = curr.right;
  }

  return result;
}

When to use iterative over recursive?

Recursive traversals are cleaner and easier to write, but they use O(h) call stack space. For very deep trees (or skewed trees where h ≈ n), iterative avoids stack overflow. In interviews, knowing both demonstrates depth of understanding.

Level-Order Traversal (BFS)

Level-order visits nodes top-to-bottom, left-to-right — one level at a time. It uses a queue instead of a stack, making it a breadth-first search.

function levelOrder<T>(root: TreeNode<T> | null): T[][] {
  if (!root) return [];

  const result: T[][] = [];
  const queue: TreeNode<T>[] = [root];

  while (queue.length) {
    const levelSize = queue.length;
    const level: T[] = [];

    for (let i = 0; i < levelSize; i++) {
      const node = queue.shift()!;
      level.push(node.val);

      if (node.left) queue.push(node.left);
      if (node.right) queue.push(node.right);
    }

    result.push(level);
  }

  return result;
}

// levelOrder(root) → [[10], [5, 15], [3, 7, 12, 20], [1]]
The "level size" trick: At the start of each iteration, queue.length is exactly the number of nodes at the current level. Process that many nodes, enqueuing their children. When the inner loop finishes, the queue contains precisely the next level.

Binary Search Tree (BST)

A BST enforces a strict ordering invariant: for every node, all values in the left subtree are smaller and all values in the right subtree are larger. This enables O(log n) search, insertion, and deletion — provided the tree stays balanced.

class BST {
  root: TreeNode<number> | null = null;

  insert(val: number): void {
    this.root = this._insert(this.root, val);
  }

  private _insert(node: TreeNode<number> | null, val: number): TreeNode<number> {
    if (!node) return new TreeNode(val);

    if (val < node.val) {
      node.left = this._insert(node.left, val);
    } else if (val > node.val) {
      node.right = this._insert(node.right, val);
    }

    return node;
  }

  search(val: number): boolean {
    let curr = this.root;
    while (curr) {
      if (val === curr.val) return true;
      curr = val < curr.val ? curr.left : curr.right;
    }
    return false;
  }

  delete(val: number): void {
    this.root = this._delete(this.root, val);
  }

  private _delete(node: TreeNode<number> | null, val: number): TreeNode<number> | null {
    if (!node) return null;

    if (val < node.val) {
      node.left = this._delete(node.left, val);
    } else if (val > node.val) {
      node.right = this._delete(node.right, val);
    } else {
      // Node found — three cases
      if (!node.left) return node.right;
      if (!node.right) return node.left;

      // Two children: replace with inorder successor
      let successor = node.right;
      while (successor.left) successor = successor.left;
      node.val = successor.val;
      node.right = this._delete(node.right, successor.val);
    }

    return node;
  }
}

Balanced vs Unbalanced

A BST only guarantees O(log n) operations when balanced. Inserting sorted data into a plain BST creates a degenerate linked list with O(n) operations. Self-balancing trees (AVL, Red-Black) maintain balance automatically, but are rarely asked in interviews — understanding the risk is enough.

Common Tree Problems

Maximum Depth

function maxDepth(root: TreeNode<number> | null): number {
  if (!root) return 0;
  return 1 + Math.max(maxDepth(root.left), maxDepth(root.right));
}

Same Tree

function isSameTree(
  p: TreeNode<number> | null,
  q: TreeNode<number> | null
): boolean {
  if (!p && !q) return true;
  if (!p || !q) return false;
  return p.val === q.val
    && isSameTree(p.left, q.left)
    && isSameTree(p.right, q.right);
}

Invert Binary Tree

function invertTree(root: TreeNode<number> | null): TreeNode<number> | null {
  if (!root) return null;

  [root.left, root.right] = [root.right, root.left];
  invertTree(root.left);
  invertTree(root.right);

  return root;
}

Subtree of Another Tree

function isSubtree(
  root: TreeNode<number> | null,
  sub: TreeNode<number> | null
): boolean {
  if (!root) return !sub;
  if (isSameTree(root, sub)) return true;
  return isSubtree(root.left, sub) || isSubtree(root.right, sub);
}

Validate BST

A common mistake is to only check that left.val < node.val < right.val. This fails because BST validity requires all left descendants to be smaller, not just the immediate child. The correct approach passes down min/max bounds.

function isValidBST(
  root: TreeNode<number> | null,
  min = -Infinity,
  max = Infinity
): boolean {
  if (!root) return true;

  if (root.val <= min || root.val >= max) return false;

  return isValidBST(root.left, min, root.val)
      && isValidBST(root.right, root.val, max);
}

// isValidBST(root) checks the entire tree
// Each recursive call narrows the valid range
Bound propagation: Going left tightens the upper bound (max = node.val). Going right tightens the lower bound (min = node.val). Any node outside its bounds invalidates the BST.

Complexity Summary

Operation / Problem Time Space
BST: search / insert / delete O(log n)* O(h)
DFS Traversal (in/pre/post) O(n) O(h)
BFS / Level-Order O(n) O(w)
Max Depth O(n) O(h)
Same Tree / Invert O(n) O(h)
Validate BST O(n) O(h)
Subtree of Another Tree O(n·m) O(h)

* O(log n) for balanced BSTs; O(n) worst-case for skewed trees. h = height, w = max width.
