Stacks

Last-In, First-Out — the data structure behind undo buttons, call stacks, and bracket matching

TypeScript LIFO

What is a Stack?

A stack is a linear data structure that follows the Last-In, First-Out (LIFO) principle. The last element added is the first one removed — like a stack of plates at a buffet. You can only add to the top and remove from the top.

Stacks are everywhere in computing:

  • Function call stack — each function call pushes a frame; returning pops it
  • Undo/Redo — each action pushes onto the undo stack; undoing pops it and pushes onto redo
  • Browser history — each page visit pushes; the back button pops
  • Expression evaluation — compilers use stacks to parse and evaluate expressions
The LIFO Contract: A stack guarantees that the most recently added element is always the first to be removed. This constraint, rather than being limiting, is precisely what makes stacks powerful for problems involving nested or recursive structure.

Stack Operations

Operation Description Time Space
push(item) Add item to the top O(1) O(1)
pop() Remove and return top item O(1) O(1)
peek() / top() View top item without removing O(1) O(1)
isEmpty() Check if stack has no elements O(1) O(1)
size() Return number of elements O(1) O(1)

Implementing a Stack in TypeScript

Array-Based Stack

The simplest approach: use a JavaScript array and restrict operations to the end. push and pop already operate on the end of an array in O(1) amortised time.

class Stack<T> {
  private items: T[] = [];

  push(item: T): void {
    this.items.push(item);
  }

  pop(): T | undefined {
    return this.items.pop();
  }

  peek(): T | undefined {
    return this.items[this.items.length - 1];
  }

  isEmpty(): boolean {
    return this.items.length === 0;
  }

  get size(): number {
    return this.items.length;
  }
}

Linked-List-Based Stack

An alternative that avoids dynamic array resizing. Each node points to the one below it. Push and pop always operate on the head — guaranteed O(1) with no amortisation needed.

interface StackNode<T> {
  value: T;
  next: StackNode<T> | null;
}

class LinkedStack<T> {
  private head: StackNode<T> | null = null;
  private _size = 0;

  push(value: T): void {
    this.head = { value, next: this.head };
    this._size++;
  }

  pop(): T | undefined {
    if (!this.head) return undefined;
    const value = this.head.value;
    this.head = this.head.next;
    this._size--;
    return value;
  }

  peek(): T | undefined {
    return this.head?.value;
  }

  isEmpty(): boolean {
    return this.head === null;
  }

  get size(): number {
    return this._size;
  }
}

Which implementation to choose?

In practice, the array-based stack is almost always faster due to CPU cache locality — contiguous memory access is much cheaper than pointer chasing. Use the linked version only when you need strict O(1) guarantees without amortisation (rare in interviews).

Valid Parentheses

The quintessential stack problem. Given a string of brackets ()[]{}, determine if every opening bracket has a matching closing bracket in the correct order. The stack naturally tracks nesting depth.

function isValid(s: string): boolean {
  const stack: string[] = [];
  const pairs: Record<string, string> = {
    ')': '(',
    ']': '[',
    '}': '{',
  };

  for (const ch of s) {
    if (ch in pairs) {
      // Closing bracket: must match the top of stack
      if (stack.pop() !== pairs[ch]) return false;
    } else {
      // Opening bracket: push onto stack
      stack.push(ch);
    }
  }

  return stack.length === 0;
}

// isValid("({[]})") → true
// isValid("([)]")  → false
// isValid("{[]")   → false (unclosed)

Min Stack

Design a stack that supports push, pop, peek, and retrieving the minimum element — all in O(1) time. The trick: maintain a parallel stack that tracks the current minimum at each level.

Why does this work? Every time we push a value, we also push the current minimum (which is either the new value or the previous minimum, whichever is smaller). When we pop, both stacks pop together, so the min stack always reflects the minimum of the remaining elements.

class MinStack {
  private stack: number[] = [];
  private mins: number[] = [];

  push(val: number): void {
    this.stack.push(val);
    const currentMin = this.mins.length === 0
      ? val
      : Math.min(val, this.mins[this.mins.length - 1]);
    this.mins.push(currentMin);
  }

  pop(): void {
    this.stack.pop();
    this.mins.pop();
  }

  top(): number {
    return this.stack[this.stack.length - 1];
  }

  getMin(): number {
    return this.mins[this.mins.length - 1];
  }
}

// const ms = new MinStack();
// ms.push(-2); ms.push(0); ms.push(-3);
// ms.getMin(); → -3
// ms.pop();
// ms.top();    → 0
// ms.getMin(); → -2
Space trade-off: We use O(n) extra space for the mins stack. This is the price of O(1) getMin. Without it, getMin would require scanning the entire stack in O(n).

Monotonic Stack

A monotonic stack maintains its elements in sorted order (either strictly increasing or decreasing). When a new element violates the order, we pop elements until the invariant is restored. This technique solves "next greater/smaller element" problems in O(n).

Next Greater Element

For each element in an array, find the first element to its right that is larger. Brute force is O(n²) — for each element, scan right. A monotonic decreasing stack solves it in a single pass.

function nextGreaterElement(nums: number[]): number[] {
  const result: number[] = new Array(nums.length).fill(-1);
  const stack: number[] = []; // stores indices

  for (let i = 0; i < nums.length; i++) {
    // Pop all indices whose values are smaller than current
    while (stack.length > 0 && nums[stack[stack.length - 1]] < nums[i]) {
      const idx = stack.pop()!;
      result[idx] = nums[i];
    }
    stack.push(i);
  }

  return result;
}

// nextGreaterElement([2, 1, 2, 4, 3])
// → [4, 2, 4, -1, -1]
// For 2: next greater is 4
// For 1: next greater is 2
// For last 3: no greater element → -1

Evaluate Reverse Polish Notation

Reverse Polish Notation (RPN) places operators after their operands: 3 4 + 2 × means (3 + 4) × 2 = 14. Evaluation is straightforward with a stack: push numbers, and when you hit an operator, pop two operands, apply the operator, and push the result.

function evalRPN(tokens: string[]): number {
  const stack: number[] = [];
  const ops: Record<string, (a: number, b: number) => number> = {
    '+': (a, b) => a + b,
    '-': (a, b) => a - b,
    '*': (a, b) => a * b,
    '/': (a, b) => Math.trunc(a / b),
  };

  for (const token of tokens) {
    if (token in ops) {
      const b = stack.pop()!;
      const a = stack.pop()!;
      stack.push(ops[token](a, b));
    } else {
      stack.push(parseInt(token));
    }
  }

  return stack[0];
}

// evalRPN(["2","1","+","3","*"]) → 9
// Evaluation: (2 + 1) * 3 = 9

// evalRPN(["4","13","5","/","+"]) → 6
// Evaluation: 4 + (13 / 5) = 4 + 2 = 6

Why Math.trunc?

The problem specifies integer division truncates toward zero. Math.trunc(-7 / 2) gives -3 (correct), while Math.floor(-7 / 2) gives -4 (rounds toward negative infinity, not what we want).
