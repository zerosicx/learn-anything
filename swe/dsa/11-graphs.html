What is a Graph?

A graph G = (V, E) is a set of vertices (nodes) connected by edges. Unlike trees, graphs can have cycles, multiple paths between nodes, and disconnected components.

Key Terminology

  • Directed vs Undirected: In a directed graph (digraph), edges have direction (A → B ≠ B → A). In undirected graphs, edges are bidirectional.
  • Weighted vs Unweighted: Edges may carry a cost/distance (weighted) or not (unweighted, implicitly weight 1).
  • Degree: Number of edges touching a node. For directed graphs: in-degree (incoming) and out-degree (outgoing).
  • Path: A sequence of vertices connected by edges. A cycle is a path that starts and ends at the same vertex.
  • Connected: An undirected graph where every vertex is reachable from every other. For directed graphs: strongly connected.
  • DAG: Directed Acyclic Graph — a directed graph with no cycles. Critical for topological sorting.
Trees are a special case of graphs — a tree is a connected, acyclic, undirected graph. Every tree with n nodes has exactly n − 1 edges. Many graph algorithms (DFS, BFS) are generalisations of tree traversals.

Graph Representations

The two standard ways to store a graph in code. The right choice depends on graph density.

Adjacency List

Map each vertex to its list of neighbours. Space-efficient for sparse graphs (most real-world graphs).

class GraphAdjList {
  private adj: Map<string, string[]> = new Map();

  addVertex(v: string): void {
    if (!this.adj.has(v)) this.adj.set(v, []);
  }

  addEdge(from: string, to: string): void {
    this.addVertex(from);
    this.addVertex(to);
    this.adj.get(from)!.push(to);
  }

  neighbours(v: string): string[] {
    return this.adj.get(v) ?? [];
  }

  vertices(): string[] {
    return [...this.adj.keys()];
  }
}

Adjacency Matrix

A 2D boolean/number array where matrix[i][j] indicates an edge from i to j. Fast O(1) edge lookup, but O(V²) space.

class GraphAdjMatrix {
  private matrix: number[][];
  private size: number;

  constructor(numVertices: number) {
    this.size = numVertices;
    this.matrix = Array.from(
      { length: numVertices },
      () => new Array(numVertices).fill(0)
    );
  }

  addEdge(from: number, to: number, weight = 1): void {
    this.matrix[from][to] = weight;
  }

  hasEdge(from: number, to: number): boolean {
    return this.matrix[from][to] !== 0;
  }

  getWeight(from: number, to: number): number {
    return this.matrix[from][to];
  }
}

Directed Graph Example

Depth-First Search (DFS)

DFS explores as deep as possible along each branch before backtracking. It uses a stack (explicit or the call stack via recursion).

Recursive DFS

function dfsRecursive(
  graph: Map<string, string[]>,
  start: string
): string[] {
  const visited = new Set<string>();
  const order: string[] = [];

  function dfs(node: string): void {
    if (visited.has(node)) return;
    visited.add(node);
    order.push(node);

    for (const neighbour of (graph.get(node) ?? [])) {
      dfs(neighbour);
    }
  }

  dfs(start);
  return order;
}

Iterative DFS (with explicit stack)

function dfsIterative(
  graph: Map<string, string[]>,
  start: string
): string[] {
  const visited = new Set<string>();
  const stack: string[] = [start];
  const order: string[] = [];

  while (stack.length > 0) {
    const node = stack.pop()!;
    if (visited.has(node)) continue;

    visited.add(node);
    order.push(node);

    for (const neighbour of (graph.get(node) ?? []).reverse()) {
      if (!visited.has(neighbour)) stack.push(neighbour);
    }
  }

  return order;
}

Why reverse neighbours?

A stack processes last-in-first-out. Reversing the neighbour list ensures we visit them in the same order as the recursive version. Without reversing, the iterative DFS visits neighbours in reverse order — functionally correct, but the traversal order differs.

Breadth-First Search (BFS)

BFS explores all neighbours at the current depth before moving deeper. It uses a queue and naturally finds the shortest path in unweighted graphs.

function bfs(
  graph: Map<string, string[]>,
  start: string
): string[] {
  const visited = new Set<string>([start]);
  const queue: string[] = [start];
  const order: string[] = [];

  while (queue.length > 0) {
    const node = queue.shift()!;
    order.push(node);

    for (const neighbour of (graph.get(node) ?? [])) {
      if (!visited.has(neighbour)) {
        visited.add(neighbour);
        queue.push(neighbour);
      }
    }
  }

  return order;
}
DFS vs BFS: DFS uses a stack, goes deep, finds any path — good for cycle detection, topological sort, and exhaustive search. BFS uses a queue, goes wide, finds the shortest path in unweighted graphs — good for shortest path, level-order traversal, and finding connected components.

Topological Sort

A topological ordering of a DAG is a linear sequence where for every edge u → v, u comes before v. Think "task ordering" — if A depends on B, B must appear first.

Kahn's Algorithm (BFS-based): Start with all zero-in-degree nodes, process them, and reduce in-degrees of their neighbours. When a neighbour hits zero, add it to the queue.

function topologicalSort(
  graph: Map<string, string[]>
): string[] {
  const inDegree = new Map<string, number>();
  for (const v of graph.keys()) inDegree.set(v, 0);

  for (const neighbours of graph.values()) {
    for (const n of neighbours) {
      inDegree.set(n, (inDegree.get(n) ?? 0) + 1);
    }
  }

  const queue: string[] = [];
  for (const [v, deg] of inDegree) {
    if (deg === 0) queue.push(v);
  }

  const order: string[] = [];

  while (queue.length > 0) {
    const node = queue.shift()!;
    order.push(node);

    for (const neighbour of (graph.get(node) ?? [])) {
      const newDeg = inDegree.get(neighbour)! - 1;
      inDegree.set(neighbour, newDeg);
      if (newDeg === 0) queue.push(neighbour);
    }
  }

  if (order.length !== graph.size) {
    throw new Error("Cycle detected — no topological ordering exists");
  }

  return order;
}

Real-world uses

Build systems (compile dependencies in order), task schedulers (prerequisites before courses), package managers (install dependencies before dependents). If a cycle is detected, the dependency graph is invalid.

Union-Find (Disjoint Set)

Union-Find tracks which elements belong to the same group. Two optimisations — union by rank and path compression — make operations nearly O(1) amortised.

class UnionFind {
  private parent: number[];
  private rank: number[];
  public components: number;

  constructor(n: number) {
    this.parent = Array.from({ length: n }, (_, i) => i);
    this.rank = new Array(n).fill(0);
    this.components = n;
  }

  find(x: number): number {
    if (this.parent[x] !== x) {
      this.parent[x] = this.find(this.parent[x]); // path compression
    }
    return this.parent[x];
  }

  union(x: number, y: number): boolean {
    const rootX = this.find(x);
    const rootY = this.find(y);
    if (rootX === rootY) return false;

    if (this.rank[rootX] < this.rank[rootY]) {
      this.parent[rootX] = rootY;
    } else if (this.rank[rootX] > this.rank[rootY]) {
      this.parent[rootY] = rootX;
    } else {
      this.parent[rootY] = rootX;
      this.rank[rootX]++;
    }
    this.components--;
    return true;
  }

  connected(x: number, y: number): boolean {
    return this.find(x) === this.find(y);
  }
}
Path compression flattens the tree so future finds are O(1). Union by rank attaches the shorter tree under the taller one. Together, they make each operation O(α(n)) — the inverse Ackermann function, effectively constant.

Number of Islands

Given a 2D grid of '1's (land) and '0's (water), count the number of islands. An island is a group of adjacent '1's connected horizontally or vertically.

function numIslands(grid: string[][]): number {
  if (grid.length === 0) return 0;
  const rows = grid.length;
  const cols = grid[0].length;
  let count = 0;

  function dfs(r: number, c: number): void {
    if (r < 0 || r >= rows || c < 0 || c >= cols) return;
    if (grid[r][c] !== "1") return;

    grid[r][c] = "0"; // mark visited by sinking

    dfs(r + 1, c);
    dfs(r - 1, c);
    dfs(r, c + 1);
    dfs(r, c - 1);
  }

  for (let r = 0; r < rows; r++) {
    for (let c = 0; c < cols; c++) {
      if (grid[r][c] === "1") {
        count++;
        dfs(r, c); // sink the entire island
      }
    }
  }

  return count;
}

//   ["0","0","1","0","0"],
//   ["0","0","0","1","1"]
// ]) → 3

Shortest Path (Dijkstra's Algorithm)

Dijkstra's finds the shortest path from a source to all other vertices in a weighted graph with non-negative edges. It greedily picks the closest unvisited vertex, then relaxes its neighbours.

function dijkstra(
  graph: Map<string, [string, number][]>,
  source: string
): Map<string, number> {
  const dist = new Map<string, number>();
  for (const v of graph.keys()) dist.set(v, Infinity);
  dist.set(source, 0);

  // Simple priority queue (min-heap would be faster for large graphs)
  const pq: [string, number][] = [[source, 0]];

  while (pq.length > 0) {
    pq.sort((a, b) => a[1] - b[1]);
    const [node, d] = pq.shift()!;

    if (d > dist.get(node)!) continue; // stale entry

    for (const [neighbour, weight] of (graph.get(node) ?? [])) {
      const newDist = d + weight;
      if (newDist < dist.get(neighbour)!) {
        dist.set(neighbour, newDist);
        pq.push([neighbour, newDist]);
      }
    }
  }

  return dist;
}

//   ["A", [["B", 4], ["C", 2]]],
//   ["B", [["D", 3], ["C", 1]]],
//   ["C", [["B", 1], ["D", 5]]],
//   ["D", []]
// ]);
// dijkstra(g, "A") → Map { A→0, B→3, C→2, D→6 }

Limitations

Dijkstra's does not work with negative edge weights — use Bellman-Ford for that. The sort-based PQ above is O(V²) total; a proper min-heap brings it to O((V + E) log V).
