Dynamic Programming

Break complex problems into overlapping subproblems, solve each once, and build optimal solutions from the ground up

TypeScript Optimisation

What is Dynamic Programming?

Dynamic programming (DP) is an algorithmic technique for solving problems that exhibit two properties:

  1. Overlapping subproblems: The same smaller problems are solved repeatedly. (Fibonacci: fib(3) appears in both fib(5) and fib(4).)
  2. Optimal substructure: The optimal solution to the problem can be constructed from optimal solutions of its subproblems.

DP eliminates redundant computation by storing results. There are two approaches:

  • Memoisation (top-down): Write the natural recursive solution, then cache results. Start from the big problem and recurse down.
  • Tabulation (bottom-up): Fill a table iteratively from the smallest subproblems up to the answer. No recursion, no stack overflow risk.
DP vs Greedy: Greedy makes the locally optimal choice at each step and never looks back. DP considers all possible choices and picks the globally optimal one. If a greedy approach works (provably), use it — it's simpler. DP is for when greedy fails.

Fibonacci — The Classic Example

The Fibonacci sequence perfectly illustrates why DP exists. The naive recursive solution recomputes the same values exponentially many times.

Naive Recursive — O(2ⁿ)

function fibNaive(n: number): number {
  if (n <= 1) return n;
  return fibNaive(n - 1) + fibNaive(n - 2);
}

Memoised (Top-Down) — O(n)

function fibMemo(
  n: number,
  memo: Map<number, number> = new Map()
): number {
  if (n <= 1) return n;
  if (memo.has(n)) return memo.get(n)!;

  const result = fibMemo(n - 1, memo) + fibMemo(n - 2, memo);
  memo.set(n, result);
  return result;
}

Tabulated (Bottom-Up) — O(n) time, O(1) space

function fibTab(n: number): number {
  if (n <= 1) return n;
  let prev2 = 0, prev1 = 1;
  for (let i = 2; i <= n; i++) {
    const curr = prev1 + prev2;
    prev2 = prev1;
    prev1 = curr;
  }
  return prev1;
}

Climbing Stairs

You can climb 1 or 2 steps at a time. How many distinct ways to reach step n? This is Fibonacci in disguise: ways(n) = ways(n-1) + ways(n-2).

Memoised

function climbStairsMemo(n: number): number {
  const memo = new Map<number, number>();

  function dp(i: number): number {
    if (i <= 1) return 1;
    if (memo.has(i)) return memo.get(i)!;
    const result = dp(i - 1) + dp(i - 2);
    memo.set(i, result);
    return result;
  }

  return dp(n);
}

Tabulated

function climbStairsTab(n: number): number {
  if (n <= 1) return 1;
  let prev2 = 1, prev1 = 1;
  for (let i = 2; i <= n; i++) {
    const curr = prev1 + prev2;
    prev2 = prev1;
    prev1 = curr;
  }
  return prev1;
}

// climbStairsTab(5)  → 8
// (paths: 11111, 1112, 1121, 1211, 2111, 122, 212, 221)

Coin Change

Given coin denominations and a target amount, find the fewest coins needed. Classic 1D tabulation: for each amount from 1 to target, try every coin and take the minimum.

function coinChange(coins: number[], amount: number): number {
  const dp = new Array(amount + 1).fill(Infinity);
  dp[0] = 0;

  for (let i = 1; i <= amount; i++) {
    for (const coin of coins) {
      if (coin <= i) {
        dp[i] = Math.min(dp[i], dp[i - coin] + 1);
      }
    }
  }

  return dp[amount] === Infinity ? -1 : dp[amount];
}

// coinChange([1, 5, 10, 25], 30)  → 2  (25 + 5)
// coinChange([2], 3)              → -1 (impossible)

DP table walkthrough for coins = [1, 3, 4], amount = 6:

Amount0123456

Best coin used — 1 1 3 4 1+4 3+3
Recurrence: dp[i] = min(dp[i - coin] + 1) for all valid coins. Each cell says "the fewest coins to make amount i." We build from 0 up, so when we compute dp[6], all smaller amounts are already solved.

Longest Common Subsequence (LCS)

Given two strings, find the length of their longest common subsequence (characters in order, not necessarily contiguous). This is a classic 2D DP problem.

function longestCommonSubsequence(
  text1: string,
  text2: string
): number {
  const m = text1.length;
  const n = text2.length;
  const dp: number[][] = Array.from(
    { length: m + 1 },
    () => new Array(n + 1).fill(0)
  );

  for (let i = 1; i <= m; i++) {
    for (let j = 1; j <= n; j++) {
      if (text1[i - 1] === text2[j - 1]) {
        dp[i][j] = dp[i - 1][j - 1] + 1;
      } else {
        dp[i][j] = Math.max(dp[i - 1][j], dp[i][j - 1]);
      }
    }
  }

  return dp[m][n];
}

// longestCommonSubsequence("abcde", "ace")  → 3  ("ace")
// longestCommonSubsequence("abc", "def")    → 0

Reading the 2D table

dp[i][j] = LCS length of text1[0..i-1] and text2[0..j-1]. If characters match, extend the diagonal. Otherwise, take the max of dropping a character from either string. Time: O(m·n). Space can be optimised to O(min(m,n)) by only keeping two rows.

0/1 Knapsack

Given items with weights and values, and a capacity limit, maximise total value without exceeding capacity. Each item is either taken or left — no fractions.

function knapsack(
  weights: number[],
  values: number[],
  capacity: number
): number {
  const n = weights.length;
  const dp: number[][] = Array.from(
    { length: n + 1 },
    () => new Array(capacity + 1).fill(0)
  );

  for (let i = 1; i <= n; i++) {
    for (let w = 0; w <= capacity; w++) {
      // Don't take item i
      dp[i][w] = dp[i - 1][w];

      // Take item i (if it fits)
      if (weights[i - 1] <= w) {
        dp[i][w] = Math.max(
          dp[i][w],
          dp[i - 1][w - weights[i - 1]] + values[i - 1]
        );
      }
    }
  }

  return dp[n][capacity];
}

// knapsack([2, 3, 4, 5], [3, 4, 5, 6], 8)  → 10  (items 2+3: weight 7, value 9… actually items 1+4: weight 7, value 9 or 2+3: wt 7, val 9)
The choice at each step: For item i with remaining capacity w, we choose the better of (1) skip item i → dp[i-1][w], or (2) take item i → dp[i-1][w - weight_i] + value_i. The i-1 row ensures each item is used at most once.

House Robber

A robber can steal from houses along a street, but can't rob two adjacent houses (alarms). Maximise total loot. This is a clean 1D DP.

function rob(nums: number[]): number {
  if (nums.length === 0) return 0;
  if (nums.length === 1) return nums[0];

  let prev2 = 0;  // dp[i-2]
  let prev1 = 0;  // dp[i-1]

  for (const num of nums) {
    const curr = Math.max(
      prev1,          // skip this house
      prev2 + num     // rob this house + best from 2 back
    );
    prev2 = prev1;
    prev1 = curr;
  }

  return prev1;
}

// rob([2, 7, 9, 3, 1])  → 12  (rob houses 0+2+4: 2+9+1=12)
// rob([1, 2, 3, 1])     → 4   (rob houses 0+2: 1+3=4)

Recurrence relation

dp[i] = max(dp[i-1], dp[i-2] + nums[i]). At each house, the robber decides: skip (keep the best from the previous house) or rob (current value + best from two houses back to avoid adjacency). Since we only need the last two values, O(1) space suffices.

Recognising DP Problems

DP problems share common language. If you see these patterns in a problem statement, think DP:

DP Signal Phrases

  • "Find the minimum/maximum..." → optimisation DP
  • "Count the number of ways..." → counting DP
  • "Is it possible to..." → feasibility DP (boolean table)
  • "Longest/shortest subsequence..." → sequence DP (often 2D)
  • "Partition into..." → subset-sum variant

DP vs Backtracking

If the problem asks you to enumerate all solutions (e.g., "list all subsets"), DP alone won't help — you need backtracking. DP shines when you need one optimal answer or a count, not the actual configurations.
