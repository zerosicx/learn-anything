Heap / Priority Queue

Efficiently access the minimum or maximum element — the backbone of priority queues, scheduling, and top-K problems

What is a Heap?

A heap is a complete binary tree stored as an array, where every parent satisfies a heap ordering property relative to its children. In a min-heap, every parent is ≤ its children; in a max-heap, every parent is ≥ its children.

"Complete binary tree" means every level is fully filled except possibly the last, which is filled left to right. This property lets us store the tree flat in an array with no wasted space — no pointers needed.

Parent/child index formulas (0-indexed):
  • Parent of i: Math.floor((i - 1) / 2)
  • Left child of i: 2 * i + 1
  • Right child of i: 2 * i + 2
These formulas are all you need to navigate the tree without any node/pointer objects.

Why does this matter? A heap gives you O(1) access to the min (or max) element while supporting O(log n) insertion and removal. This makes it the natural choice whenever you need to repeatedly ask "what's the most urgent item?" — task schedulers, Dijkstra's algorithm, median-finding, and merge-K-sorted-anything problems.

Implementing a MinHeap in TypeScript

The two core operations are bubble up (after insertion) and bubble down (after extraction). Every other operation builds on these.

class MinHeap {
  private data: number[] = [];

  get size(): number {
    return this.data.length;
  }

  peek(): number | undefined {
    return this.data[0];
  }

  insert(val: number): void {
    this.data.push(val);
    this.bubbleUp(this.data.length - 1);
  }

  extractMin(): number | undefined {
    if (this.data.length === 0) return undefined;
    const min = this.data[0];
    const last = this.data.pop()!;
    if (this.data.length > 0) {
      this.data[0] = last;
      this.bubbleDown(0);
    }
    return min;
  }

  private bubbleUp(i: number): void {
    while (i > 0) {
      const parent = Math.floor((i - 1) / 2);
      if (this.data[i] >= this.data[parent]) break;
      [this.data[i], this.data[parent]] = [this.data[parent], this.data[i]];
      i = parent;
    }
  }

  private bubbleDown(i: number): void {
    const n = this.data.length;
    while (true) {
      let smallest = i;
      const left = 2 * i + 1;
      const right = 2 * i + 2;

      if (left < n && this.data[left] < this.data[smallest]) smallest = left;
      if (right < n && this.data[right] < this.data[smallest]) smallest = right;
      if (smallest === i) break;

      [this.data[i], this.data[smallest]] = [this.data[smallest], this.data[i]];
      i = smallest;
    }
  }
}

Bubble Up vs Bubble Down

Bubble up moves a node upward by swapping with its parent until the heap property holds — used after insertion at the bottom. Bubble down moves a node downward by swapping with its smallest child — used after placing the last element at the root during extraction. Both are O(log n) since the tree height is log n.

MaxHeap

A max-heap is identical to a min-heap with the comparison flipped. Rather than duplicating the entire class, we can parameterise with a comparator — but for clarity, here's the minimal change:

class MaxHeap {
  private data: number[] = [];

  insert(val: number): void {
    this.data.push(val);
    this.bubbleUp(this.data.length - 1);
  }

  extractMax(): number | undefined {
    if (this.data.length === 0) return undefined;
    const max = this.data[0];
    const last = this.data.pop()!;
    if (this.data.length > 0) {
      this.data[0] = last;
      this.bubbleDown(0);
    }
    return max;
  }

  private bubbleUp(i: number): void {
    while (i > 0) {
      const parent = Math.floor((i - 1) / 2);
      if (this.data[i] <= this.data[parent]) break; // ← flipped
      [this.data[i], this.data[parent]] = [this.data[parent], this.data[i]];
      i = parent;
    }
  }

  private bubbleDown(i: number): void {
    const n = this.data.length;
    while (true) {
      let largest = i;
      const left = 2 * i + 1;
      const right = 2 * i + 2;

      if (left < n && this.data[left] > this.data[largest]) largest = left;
      if (right < n && this.data[right] > this.data[largest]) largest = right;
      if (largest === i) break;

      [this.data[i], this.data[largest]] = [this.data[largest], this.data[i]];
      i = largest;
    }
  }
}

Generic Heap

In production code, use a single Heap<T> class that accepts a comparator: (a: T, b: T) => number. Pass (a, b) => a - b for min-heap, (a, b) => b - a for max-heap. This avoids code duplication and supports custom types.

Heap Sort

Heap sort works in two phases: build a max-heap from the input, then repeatedly extract the max and place it at the end. It achieves O(n log n) worst-case time and O(1) extra space (in-place).

function heapSort(arr: number[]): number[] {
  const n = arr.length;

  function siftDown(i: number, size: number): void {
    while (true) {
      let largest = i;
      const left = 2 * i + 1;
      const right = 2 * i + 2;
      if (left < size && arr[left] > arr[largest]) largest = left;
      if (right < size && arr[right] > arr[largest]) largest = right;
      if (largest === i) break;
      [arr[i], arr[largest]] = [arr[largest], arr[i]];
      i = largest;
    }
  }

  // Phase 1: Build max-heap (bottom-up)
  for (let i = Math.floor(n / 2) - 1; i >= 0; i--) {
    siftDown(i, n);
  }

  // Phase 2: Extract max repeatedly
  for (let end = n - 1; end > 0; end--) {
    [arr[0], arr[end]] = [arr[end], arr[0]];
    siftDown(0, end);
  }

  return arr;
}

// heapSort([4, 10, 3, 5, 1])  → [1, 3, 4, 5, 10]
Why build the heap bottom-up? Calling siftDown from the last non-leaf node backwards is O(n), not O(n log n). Leaf nodes (roughly half the array) need no work. Nodes one level up need at most 1 swap. This "heapify" trick is faster than inserting n elements one by one.

Top K Elements Pattern

Finding the k largest elements in an array is a classic heap problem. The key insight: maintain a min-heap of size k. For every new element, if it's larger than the heap's minimum, swap it in. At the end, the heap contains exactly the k largest elements.

function topKLargest(nums: number[], k: number): number[] {
  const heap = new MinHeap();

  for (const num of nums) {
    heap.insert(num);
    if (heap.size > k) {
      heap.extractMin(); // evict smallest — only k largest remain
    }
  }

  const result: number[] = [];
  while (heap.size > 0) {
    result.push(heap.extractMin()!);
  }
  return result;
}

// topKLargest([3, 1, 5, 12, 2, 11], 3)  → [5, 11, 12]

Why min-heap for largest?

It sounds backwards, but the min-heap acts as a gatekeeper. Its root is the smallest of the k largest seen so far. Any new element must beat that threshold to enter the top-k club. Time: O(n log k). Space: O(k).

Merge K Sorted Lists

Given k sorted arrays, merge them into one sorted array. Use a min-heap to always pick the globally smallest among the k current heads.

interface HeapEntry {
  val: number;
  listIdx: number;
  elemIdx: number;
}

function mergeKSorted(lists: number[][]): number[] {
  const result: number[] = [];

  // Simple heap using array + comparator
  const heap: HeapEntry[] = [];

  function push(entry: HeapEntry): void {
    heap.push(entry);
    let i = heap.length - 1;
    while (i > 0) {
      const p = Math.floor((i - 1) / 2);
      if (heap[i].val >= heap[p].val) break;
      [heap[i], heap[p]] = [heap[p], heap[i]];
      i = p;
    }
  }

  function pop(): HeapEntry {
    const top = heap[0];
    const last = heap.pop()!;
    if (heap.length > 0) {
      heap[0] = last;
      let i = 0;
      while (true) {
        let s = i, l = 2*i+1, r = 2*i+2;
        if (l < heap.length && heap[l].val < heap[s].val) s = l;
        if (r < heap.length && heap[r].val < heap[s].val) s = r;
        if (s === i) break;
        [heap[i], heap[s]] = [heap[s], heap[i]];
        i = s;
      }
    }
    return top;
  }

  // Seed heap with first element from each list
  for (let i = 0; i < lists.length; i++) {
    if (lists[i].length > 0) {
      push({ val: lists[i][0], listIdx: i, elemIdx: 0 });
    }
  }

  while (heap.length > 0) {
    const { val, listIdx, elemIdx } = pop();
    result.push(val);
    if (elemIdx + 1 < lists[listIdx].length) {
      push({ val: lists[listIdx][elemIdx + 1], listIdx, elemIdx: elemIdx + 1 });
    }
  }

  return result;
}

// mergeKSorted([[1,4,7], [2,5,8], [3,6,9]])
// → [1, 2, 3, 4, 5, 6, 7, 8, 9]
Complexity: With k lists and n total elements, the heap never exceeds size k. Each of the n elements is pushed and popped once → O(n log k) time, O(k) space.

Complexity Summary

Operation Time Notes
peek (min/max) O(1) Root of the array
insert O(log n) Append + bubble up
extractMin/Max O(log n) Swap root with last + bubble down
heapify (build) O(n) Bottom-up sift down
heap sort O(n log n) Build + n extractions
top-k O(n log k) Heap of size k
merge k sorted O(n log k) n total elements, heap of size k
Space O(n) Array-backed, no pointers
