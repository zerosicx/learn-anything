Bitwise Operators

Bitwise operators work on individual bits of a number's binary representation. They're the fastest operations a CPU can perform — single-clock-cycle instructions that can replace more expensive arithmetic in the right context.

OperatorSymbolDescriptionExample (5 = 0101, 3 = 0011)
AND&1 if both bits are 15 & 3 = 1 (0001)
OR|1 if either bit is 15 | 3 = 7 (0111)
XOR^1 if bits differ5 ^ 3 = 6 (0110)
NOT~Flip all bits~5 = -6 (two's complement)
Left Shift<<Shift left, fill with 0s5 << 1 = 10 (1010)
Right Shift>>Shift right (arithmetic)5 >> 1 = 2 (0010)
Unsigned Right Shift>>>Shift right, fill with 0s-1 >>> 0 = 4294967295

TypeScript caveat

JavaScript (and TypeScript) bitwise operators convert numbers to 32-bit signed integers before operating. This means bitwise operations on numbers > 2³¹-1 or < -2³¹ may give unexpected results. For most interview problems dealing with 32-bit integers, this is fine.

Binary Representation

Every integer is stored as a sequence of bits. Understanding binary is essential for bit manipulation problems. In a 32-bit signed integer, bit 31 is the sign bit (0 = positive, 1 = negative using two's complement).

// View binary representation in TypeScript
const n = 42;
console.log("??", n.toString(2));
// "101010"

// Pad to see all 32 bits
function toBin32(n: number): string {
  return (n >>> 0).toString(2).padStart(32, '0');
}

console.log("??", toBin32(42));
// "00000000000000000000000000101010"

console.log("??", toBin32(-1));
// "11111111111111111111111111111111" (all 1s = -1 in two's complement)

Common Bit Tricks

These one-liners appear constantly in bit manipulation problems. Each exploits a specific property of binary arithmetic.

Check Even / Odd

const isOdd = (n: number): boolean => (n & 1) === 1;
// Last bit is 1 for odd, 0 for even. Faster than n % 2.

Check Power of 2

const isPowerOf2 = (n: number): boolean =>
  n > 0 && (n & (n - 1)) === 0;
// Powers of 2 have exactly one set bit: 1000...0
// n-1 flips all bits from that bit down: 0111...1
// AND gives 0 only if there was exactly one set bit.

Swap Without Temp Variable

function bitwiseSwap(a: number, b: number): [number, number] {
  a = a ^ b;
  b = a ^ b;  // b = (a^b)^b = a
  a = a ^ b;  // a = (a^b)^a = b
  return [a, b];
}
// XOR is its own inverse: x ^ x = 0, x ^ 0 = x

Toggle / Set / Clear a Specific Bit

const setBit = (n: number, pos: number): number =>
  n | (1 << pos);

const clearBit = (n: number, pos: number): number =>
  n & ~(1 << pos);

const toggleBit = (n: number, pos: number): number =>
  n ^ (1 << pos);

const getBit = (n: number, pos: number): number =>
  (n >> pos) & 1;

Single Number

Given an array where every element appears twice except one, find the unique element. XOR cancels out duplicates because a ^ a = 0 and a ^ 0 = a.

function singleNumber(nums: number[]): number {
  let result = 0;
  for (const num of nums) {
    result ^= num;
  }
  return result;
}

// singleNumber([4, 1, 2, 1, 2])  → 4
// Walkthrough: 0^4=4, 4^1=5, 5^2=7, 7^1=6, 6^2=4 ✓
XOR properties that make this work:
  • Commutative: a ^ b = b ^ a — order doesn't matter.
  • Associative: (a ^ b) ^ c = a ^ (b ^ c) — grouping doesn't matter.
  • Self-inverse: a ^ a = 0 — pairs cancel out.
  • Identity: a ^ 0 = a — XOR with zero is identity.

So XORing all elements groups pairs into a ^ a = 0, leaving only the unique element.

Number of 1 Bits

Count the number of set bits (1s) in a number's binary representation. Brian Kernighan's algorithm: n & (n - 1) clears the lowest set bit. Count how many times we can do this before reaching zero.

function hammingWeight(n: number): number {
  let count = 0;
  while (n !== 0) {
    n = n & (n - 1);
    count++;
  }
  return count;
}

// hammingWeight(11)  → 3   (1011 has three 1-bits)
// hammingWeight(128) → 1   (10000000)
// hammingWeight(255) → 8   (11111111)

Why n & (n-1) clears the lowest set bit

Subtracting 1 from a number flips all bits from the lowest set bit downward. For example: 1010 - 1 = 1001. ANDing with the original zeroes out that lowest 1-bit and everything below, while preserving everything above. Runs in O(k) where k is the number of set bits — faster than checking all 32 bits.

Counting Bits

For every number from 0 to n, count the number of 1-bits. The DP insight: countBits[i] = countBits[i >> 1] + (i & 1). Shifting right removes the last bit, and we add 1 if that last bit was set.

function countBits(n: number): number[] {
  const result = new Array(n + 1).fill(0);

  for (let i = 1; i <= n; i++) {
    result[i] = result[i >> 1] + (i & 1);
  }

  return result;
}

// countBits(5)  → [0, 1, 1, 2, 1, 2]
//                  0  1  10 11 100 101
// countBits(2)  → [0, 1, 1]

Time: O(n). Space: O(n). Each number's bit count is computed in O(1) from a previously computed value.

Reverse Bits

Reverse all 32 bits of a given unsigned integer. Process bit by bit: extract the lowest bit, place it in the reversed position.

function reverseBits(n: number): number {
  let result = 0;

  for (let i = 0; i < 32; i++) {
    result = (result << 1) | (n & 1);
    n >>>= 1;
  }

  return result >>> 0;
}

// reverseBits(43261596)
// Binary: 00000010100101000001111010011100
// Reversed:00111001011110000010100101000000
// → 964176192

The >>> 0 at the end converts the result to an unsigned 32-bit integer, which matters when the reversed number would set the sign bit.

Missing Number

Given an array containing n distinct numbers from the range [0, n], find the one missing. XOR all numbers with all indices — duplicates cancel, leaving the missing number.

function missingNumber(nums: number[]): number {
  let xor = nums.length;

  for (let i = 0; i < nums.length; i++) {
    xor ^= i ^ nums[i];
  }

  return xor;
}

// missingNumber([3, 0, 1])     → 2
// missingNumber([9,6,4,2,3,5,7,0,1]) → 8
// missingNumber([0])           → 1

Alternative: sum formula

You could also use n*(n+1)/2 - sum(nums). Both are O(n) time, O(1) space. The XOR approach avoids any risk of integer overflow (relevant in other languages; in JS both are safe for reasonable n).
