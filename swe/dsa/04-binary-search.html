Binary Search

Halving the search space with every comparison — the logarithmic leap from O(n) to O(log n)

TypeScript O(log n)

The Binary Search Algorithm

Binary search is deceptively simple: given a sorted collection, repeatedly halve the search space by comparing the target to the middle element. If the target is smaller, search the left half; if larger, search the right half. Each step eliminates half of the remaining candidates.

This halving is what gives binary search its O(log n) time complexity. For an array of 1 billion elements, binary search needs at most ~30 comparisons. Linear search would need up to 1 billion.

Prerequisite: Binary search requires the data to be sorted (or at least monotonically partitioned). If you find yourself needing repeated searches on unsorted data, sort first — one O(n log n) sort enables unlimited O(log n) lookups.

Basic Binary Search

The standard implementation: maintain a low and high boundary, compute mid, compare, and shrink the window.

function binarySearch(nums: number[], target: number): number {
  let low = 0;
  let high = nums.length - 1;

  while (low <= high) {
    // Avoids integer overflow compared to (low + high) / 2
    const mid = low + Math.floor((high - low) / 2);

    if (nums[mid] === target) {
      return mid;
    }

    if (nums[mid] < target) {
      low = mid + 1;   // Target is in the right half
    } else {
      high = mid - 1;  // Target is in the left half
    }
  }

  return -1; // Not found
}

The Off-by-One Minefield

Binary search has famously subtle boundary conditions. The three critical decisions:

  • Loop condition: low <= high (inclusive) vs low < high (exclusive on right)
  • Mid update: mid + 1 and mid - 1 — never use just mid as the new boundary, or you risk infinite loops
  • Mid calculation: low + Math.floor((high - low) / 2) prevents overflow in languages with fixed-size integers

Binary Search Variations

Find First Occurrence

Standard binary search finds any occurrence. To find the first (leftmost), continue searching left even after finding a match.

function findFirst(nums: number[], target: number): number {
  let low = 0;
  let high = nums.length - 1;
  let result = -1;

  while (low <= high) {
    const mid = low + Math.floor((high - low) / 2);

    if (nums[mid] === target) {
      result = mid;
      high = mid - 1;  // Keep searching left for earlier occurrence
    } else if (nums[mid] < target) {
      low = mid + 1;
    } else {
      high = mid - 1;
    }
  }

  return result;
}

Find Last Occurrence

function findLast(nums: number[], target: number): number {
  let low = 0;
  let high = nums.length - 1;
  let result = -1;

  while (low <= high) {
    const mid = low + Math.floor((high - low) / 2);

    if (nums[mid] === target) {
      result = mid;
      low = mid + 1;  // Keep searching right for later occurrence
    } else if (nums[mid] < target) {
      low = mid + 1;
    } else {
      high = mid - 1;
    }
  }

  return result;
}

Search in Rotated Sorted Array

A sorted array that's been rotated (e.g., [4,5,6,7,0,1,2]) has two sorted halves. At each step, determine which half is sorted and whether the target lies within it.

function searchRotated(nums: number[], target: number): number {
  let low = 0;
  let high = nums.length - 1;

  while (low <= high) {
    const mid = low + Math.floor((high - low) / 2);

    if (nums[mid] === target) return mid;

    // Left half is sorted
    if (nums[low] <= nums[mid]) {
      if (nums[low] <= target && target < nums[mid]) {
        high = mid - 1;
      } else {
        low = mid + 1;
      }
    }
    // Right half is sorted
    else {
      if (nums[mid] < target && target <= nums[high]) {
        low = mid + 1;
      } else {
        high = mid - 1;
      }
    }
  }

  return -1;
}

// searchRotated([4,5,6,7,0,1,2], 0) → 4

Search a 2D Matrix

Given an m×n matrix where each row is sorted and the first element of each row is greater than the last element of the previous row, search for a target value. The key insight: treat the 2D matrix as a virtual 1D sorted array.

function searchMatrix(matrix: number[][], target: number): boolean {
  const m = matrix.length;
  const n = matrix[0].length;
  let low = 0;
  let high = m * n - 1;

  while (low <= high) {
    const mid = low + Math.floor((high - low) / 2);

    // Convert 1D index to 2D coordinates
    const row = Math.floor(mid / n);
    const col = mid % n;
    const value = matrix[row][col];

    if (value === target) return true;

    if (value < target) {
      low = mid + 1;
    } else {
      high = mid - 1;
    }
  }

  return false;
}

// searchMatrix([[1,3,5],[10,11,13],[23,30,34]], 13) → true
// Virtual 1D: [1,3,5,10,11,13,23,30,34]
// Index 5 → row=5/3=1, col=5%3=2 → matrix[1][2] = 13 ✓
1D ↔ 2D mapping: For a matrix with n columns, a virtual index i maps to row = Math.floor(i / n) and col = i % n. This trick appears in many matrix problems.

Binary Search on Answer

A powerful but less obvious application: instead of searching for an element in an array, binary search the answer space. If you can frame a problem as "what is the minimum/maximum value X such that condition(X) is true?", and condition(X) is monotonic, you can binary search X.

Koko Eating Bananas

Koko has piles of bananas and h hours to eat them all. She picks a speed k (bananas per hour) and eats one pile at a time, taking ceil(pile / k) hours per pile. Find the minimum k so she finishes within h hours.

function minEatingSpeed(piles: number[], h: number): number {
  // The answer is somewhere between 1 and max(piles)
  let low = 1;
  let high = Math.max(...piles);

  while (low < high) {
    const mid = low + Math.floor((high - low) / 2);

    if (canFinish(piles, mid, h)) {
      high = mid;      // mid works, try slower
    } else {
      low = mid + 1;   // mid too slow, go faster
    }
  }

  return low;
}

function canFinish(piles: number[], speed: number, hours: number): boolean {
  let total = 0;
  for (const pile of piles) {
    total += Math.ceil(pile / speed);
  }
  return total <= hours;
}

// minEatingSpeed([3, 6, 7, 11], 8) → 4
// At speed 4: ceil(3/4)+ceil(6/4)+ceil(7/4)+ceil(11/4)
//           = 1 + 2 + 2 + 3 = 8 hours ✓

How to Spot "Binary Search on Answer"

Look for problems that ask for a minimum or maximum value and where you can write a canAchieve(x) function that returns true/false. If canAchieve flips from false to true (or true to false) at some threshold, binary search finds that threshold.
