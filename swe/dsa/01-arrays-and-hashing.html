Arrays & Hashing

Contiguous memory, index-based access, and the data structures that power everything else

TypeScript Fundamentals

What is an Array?

An array is the simplest and most fundamental data structure: a contiguous block of memory where each element is stored at a fixed offset from the start. Because elements sit side-by-side in memory, you can jump to any index in constant time — the CPU just calculates base_address + index × element_size.

This direct-access property is what makes arrays so powerful. No traversal required, no pointer chasing — just arithmetic.

Key Insight: Array access is O(1) because the memory address of any element can be computed directly from the index. This is the foundation for hash maps, heaps, and many other structures.

Dynamic Arrays in TypeScript

In lower-level languages like C, arrays have a fixed size — you allocate a block of memory and that's all you get. JavaScript (and by extension TypeScript) gives you dynamic arrays that resize automatically.

Under the hood, when you push beyond the current capacity, the engine allocates a new, larger block of memory (typically 2× the old size), copies all elements over, and frees the old block. This means most push operations are O(1), but occasionally one is O(n) due to the copy. Amortised, it averages out to O(1).

// Creating and manipulating arrays in TypeScript
const nums: number[] = [1, 2, 3, 4, 5];

// Push — append to end: amortised O(1)
nums.push(6);        // [1, 2, 3, 4, 5, 6]

// Pop — remove from end: O(1)
nums.pop();          // [1, 2, 3, 4, 5]

// Unshift — insert at start: O(n) — every element shifts right
nums.unshift(0);     // [0, 1, 2, 3, 4, 5]

// Shift — remove from start: O(n) — every element shifts left
nums.shift();        // [1, 2, 3, 4, 5]

Watch Out

unshift and shift are O(n) because every element after the insertion/removal point must be moved. If you need frequent insertions at the front, consider a deque or linked list instead.

Hash Maps

A hash map (also called a hash table or dictionary) stores key-value pairs with near-constant-time lookup. The idea is simple: take a key, run it through a hash function to produce a numeric index, and store the value at that index in an underlying array.

The hash function's job is to distribute keys uniformly across buckets. When two keys hash to the same index — a collision — we need a strategy. The most common is chaining: each bucket holds a linked list (or array) of all key-value pairs that hashed to that index.

Average-case O(1): With a good hash function and a reasonable load factor, lookups, insertions, and deletions are all O(1) on average. Worst case (all keys collide into one bucket) degrades to O(n), which is why hash function quality matters.

Implementing a HashMap in TypeScript

Let's build a hash map from scratch using chaining for collision resolution. Each bucket is an array of key-value pairs. When the load factor exceeds a threshold, we resize.

class HashMap<K, V> {
  private buckets: [K, V][][];
  private _size: number = 0;
  private readonly LOAD_FACTOR = 0.75;

  constructor(capacity = 16) {
    this.buckets = Array.from({ length: capacity }, () => []);
  }

  get size(): number {
    return this._size;
  }

  private hash(key: K): number {
    const str = String(key);
    let h = 0;
    for (let i = 0; i < str.length; i++) {
      h = (h << 5) - h + str.charCodeAt(i);
      h |= 0; // Convert to 32-bit integer
    }
    return Math.abs(h) % this.buckets.length;
  }

  set(key: K, value: V): void {
    const idx = this.hash(key);
    const bucket = this.buckets[idx];

    for (const pair of bucket) {
      if (pair[0] === key) {
        pair[1] = value;
        return;
      }
    }

    bucket.push([key, value]);
    this._size++;

    if (this._size / this.buckets.length > this.LOAD_FACTOR) {
      this.resize();
    }
  }

  get(key: K): V | undefined {
    const bucket = this.buckets[this.hash(key)];
    for (const [k, v] of bucket) {
      if (k === key) return v;
    }
    return undefined;
  }

  has(key: K): boolean {
    return this.get(key) !== undefined;
  }

  delete(key: K): boolean {
    const idx = this.hash(key);
    const bucket = this.buckets[idx];
    const i = bucket.findIndex(([k]) => k === key);
    if (i === -1) return false;
    bucket.splice(i, 1);
    this._size--;
    return true;
  }

  private resize(): void {
    const old = this.buckets;
    this.buckets = Array.from({ length: old.length * 2 }, () => []);
    this._size = 0;
    for (const bucket of old) {
      for (const [k, v] of bucket) {
        this.set(k, v);
      }
    }
  }
}

Hash Sets

A hash set is like a hash map but stores only keys — no values. It answers one question: "Is this element in the set?" in O(1) average time. TypeScript provides a built-in Set that works exactly this way.

const seen = new Set<number>();

seen.add(1);         // Set { 1 }
seen.add(2);         // Set { 1, 2 }
seen.add(1);         // Set { 1, 2 } — duplicates ignored

seen.has(2);         // true  — O(1) lookup
seen.delete(1);      // true  — O(1) removal
seen.size;           // 1

// Common use: deduplication
const unique = [...new Set([1, 2, 2, 3, 3, 3])]; // [1, 2, 3]

When to use Set vs Map

Use a Set when you only care about membership ("have I seen this before?"). Use a Map when you need to associate a value with each key ("how many times have I seen this?").

Key Patterns

Frequency Counting

The most common use of a hash map in interview problems: count how many times each element appears. This unlocks problems like "find duplicates", "top K frequent elements", and "valid anagram".

function frequencyCount(arr: number[]): Map<number, number> {
  const freq = new Map<number, number>();
  for (const num of arr) {
    freq.set(num, (freq.get(num) ?? 0) + 1);
  }
  return freq;
}

// Example: are two strings anagrams?
function isAnagram(s: string, t: string): boolean {
  if (s.length !== t.length) return false;
  const count = new Map<string, number>();
  for (const c of s) count.set(c, (count.get(c) ?? 0) + 1);
  for (const c of t) {
    const val = count.get(c);
    if (!val) return false;
    count.set(c, val - 1);
  }
  return true;
}

Two Sum Pattern

The classic: given an array and a target, find two numbers that add up to the target. The brute-force approach checks every pair in O(n²). A hash map reduces this to O(n) by storing complements as we iterate.

function twoSum(nums: number[], target: number): [number, number] | null {
  const seen = new Map<number, number>(); // value → index

  for (let i = 0; i < nums.length; i++) {
    const complement = target - nums[i];
    const j = seen.get(complement);

    if (j !== undefined) {
      return [j, i];
    }
    seen.set(nums[i], i);
  }

  return null;
}

// twoSum([2, 7, 11, 15], 9) → [0, 1]
// Because nums[0] + nums[1] = 2 + 7 = 9
The Complement Trick: Instead of asking "which two numbers sum to target?", ask "for each number, is its complement (target − number) already in my map?" This converts a nested loop into a single pass.
