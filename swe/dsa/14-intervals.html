Interval Problems

An interval [start, end] represents a range on a number line — a meeting time, a process lifetime, a booking window. Interval problems ask you to merge overlapping ranges, count conflicts, or find gaps.

The universal first step is sort by start time. This converts a chaotic set of intervals into a sequence you can process left-to-right in a single sweep. Nearly every interval problem follows this pattern:

  1. Sort intervals by their start (or sometimes end).
  2. Iterate once, maintaining some state (current merged interval, count of overlaps, etc.).
  3. Make greedy decisions based on overlap detection: do intervals a and b overlap? Only if a.end >= b.start (assuming sorted by start).

Merge Intervals

Given a list of intervals, merge all overlapping intervals. Sort by start, then sweep through and extend the current interval whenever the next one overlaps.

function merge(intervals: number[][]): number[][] {
  intervals.sort((a, b) => a[0] - b[0]);
  const merged: number[][] = [intervals[0]];

  for (let i = 1; i < intervals.length; i++) {
    const last = merged[merged.length - 1];
    const curr = intervals[i];

    if (curr[0] <= last[1]) {
      // Overlapping — extend the end
      last[1] = Math.max(last[1], curr[1]);
    } else {
      merged.push(curr);
    }
  }

  return merged;
}

// merge([[1,3],[2,6],[8,10],[15,18]])
// → [[1,6],[8,10],[15,18]]

Insert Interval

Given a sorted, non-overlapping list of intervals and a new interval, insert it and merge if necessary. Partition into three groups: intervals entirely before, intervals that overlap, and intervals entirely after.

function insert(
  intervals: number[][],
  newInterval: number[]
): number[][] {
  const result: number[][] = [];
  let i = 0;

  // 1. Add all intervals that end before newInterval starts
  while (i < intervals.length && intervals[i][1] < newInterval[0]) {
    result.push(intervals[i]);
    i++;
  }

  // 2. Merge all overlapping intervals with newInterval
  while (i < intervals.length && intervals[i][0] <= newInterval[1]) {
    newInterval[0] = Math.min(newInterval[0], intervals[i][0]);
    newInterval[1] = Math.max(newInterval[1], intervals[i][1]);
    i++;
  }
  result.push(newInterval);

  // 3. Add remaining intervals
  while (i < intervals.length) {
    result.push(intervals[i]);
    i++;
  }

  return result;
}

// insert([[1,3],[6,9]], [2,5])  → [[1,5],[6,9]]
// insert([[1,2],[3,5],[6,7],[8,10],[12,16]], [4,8])
// → [[1,2],[3,10],[12,16]]

Non-Overlapping Intervals

Given intervals, find the minimum number of intervals to remove so the rest don't overlap. Equivalent to: find the maximum number of non-overlapping intervals (classic activity selection).

Greedy strategy: sort by end time. Always keep the interval that finishes earliest — it leaves the most room for future intervals.

function eraseOverlapIntervals(
  intervals: number[][]
): number {
  intervals.sort((a, b) => a[1] - b[1]);

  let kept = 1;
  let lastEnd = intervals[0][1];

  for (let i = 1; i < intervals.length; i++) {
    if (intervals[i][0] >= lastEnd) {
      kept++;
      lastEnd = intervals[i][1];
    }
  }

  return intervals.length - kept;
}

// eraseOverlapIntervals([[1,2],[2,3],[3,4],[1,3]])  → 1  (remove [1,3])
// eraseOverlapIntervals([[1,2],[1,2],[1,2]])        → 2
Activity Selection: This is the classic activity selection problem. Sort by end time, greedily pick non-conflicting activities. The proof: by choosing the earliest-ending activity, you never do worse than any other choice because it maximises the remaining time for future activities.

Meeting Rooms

Given meeting time intervals, determine if a person can attend all meetings (i.e., no two overlap).

function canAttendMeetings(
  intervals: number[][]
): boolean {
  intervals.sort((a, b) => a[0] - b[0]);

  for (let i = 1; i < intervals.length; i++) {
    if (intervals[i][0] < intervals[i - 1][1]) {
      return false;
    }
  }

  return true;
}

// canAttendMeetings([[0,30],[5,10],[15,20]])  → false
// canAttendMeetings([[7,10],[2,4]])            → true

Meeting Rooms II

Find the minimum number of conference rooms needed. Use a sweep line: track when meetings start and end as events. At any point in time, the number of active meetings is the number of rooms needed.

function minMeetingRooms(
  intervals: number[][]
): number {
  const starts = intervals.map(i => i[0]).sort((a, b) => a - b);
  const ends = intervals.map(i => i[1]).sort((a, b) => a - b);

  let rooms = 0;
  let maxRooms = 0;
  let s = 0, e = 0;

  while (s < starts.length) {
    if (starts[s] < ends[e]) {
      rooms++;
      maxRooms = Math.max(maxRooms, rooms);
      s++;
    } else {
      rooms--;
      e++;
    }
  }

  return maxRooms;
}

// minMeetingRooms([[0,30],[5,10],[15,20]])  → 2
// minMeetingRooms([[7,10],[2,4]])            → 1

Sweep line technique

Think of a vertical line sweeping left to right across the timeline. Each meeting start adds +1, each end adds -1. The maximum value at any point is the answer. Sorting starts and ends separately gives us O(n log n) time and O(n) space — simpler and faster than maintaining a min-heap.

Interval Patterns Summary

The Sort-Then-Sweep Template

  1. Sort by start time (or end time for activity selection).
  2. Initialise tracking state (current interval, count, etc.).
  3. Sweep left to right, comparing each interval against the tracked state.
  4. Overlap check: curr.start <= prev.end means overlap.
  5. Merge: extend prev.end = max(prev.end, curr.end).
  6. No overlap: push the current interval and update tracking.

When to sort by end instead of start: When you want to maximise the number of non-overlapping intervals (activity selection / minimum removals).
